Hi! I'm Richard Grunert, a polyglot, multi-paradigm programmer who's eager to build amazing software and write beautiful code. I'm a passionate developer who's worked in multiple languages and with a number of very powerful tools -- and now I want to build something for you!

I thrive on facing new challenges and always relish the chance to learn a new tool or framework. Each project is a new opportunity to expand my skillset, and I'm a very fast learner who can quickly make sense of vast technological complexity. While I personally prefer functional programming languages, I'm also very comfortable working in an object-oriented style.

Most of my work thus far can be categorized into three main areas: full-stack web development, Android development and Linux server administration.

As a full-stack web developer, I've built complex websites featuring user accounts, custom RESTful interfaces, complex database schemas and more. Much of my work has been focused on maintaining Accident Data Center, a Drupal site that averages about 30,000 visits per month. I also designed and built Malbum, which is a bit like a private Instagram you can deploy yourself.

As an Android developer, I wrote and published the game Lunar Invasion in 2015. I've also worked on a host of smaller apps which I've written for my own self-education, including a native Android client for Malbum capable of displaying and uploading images from phone to server and vice-versa.

As a Linux administrator, I've set up, maintained and repaired servers and desktops running a variety of different distributions. I run my own virtual private server (which this site is running on!) and am very comfortable on the command-line and with shell scripts. I'm proficient with high-end web servers, and deploying open source software is a breeze. I also work with Windows and OSX machines on a daily basis.

The thing I want to emphasize most is that I'm always very eager to broaden my skillset. Check out the rest of this site to get an idea of things I've worked on in the past, or, better yet, check out my resume or GitHub.

Give me the chance to write your code and I promise you wont be disappointed!

I graduated with a Bachelor's from Western Washington University in 2013. I'm also a published writer who's written for a large metropolitan weekly.

I've worked in...

Java

My best language! I've written the more Java code than anything else, and use it primarily to write Android apps. Recently, I've been having a lot of fun with Java 8's features -- lambdas and streams are a lot of fun!

Clojure

My favorite language! Clojure is a dialect of Lisp that runs on the JVM and compiles to Java bytecode. I think programming in Lisp is great fun and use it mostly for small, quick projects when I don't want to deal with the syntactical verbosity of Java. This site's software is written in Clojure.

PHP

My first language! PHP is how I got started doing web development, and while I prefer other languages today, I still need to dive back into it every once in a while to debug existing code. My first ever project, Athena, was written in PHP, as well as a custom Drupal module I wrote for Accident Data Center.

Python

Another language I use for small, quick scripts. I've mostly used Python to write custom web spiders using the Scrapy framework.

JavaScript

An impossible-to-escape language for any web developer, I use JS to write front-end functionality for web pages I create. When possible, I prefer to use the jQuery library in place of raw JS, but I'm just as capable of the raw stuff too.

C/C++

I'm very comfortable reading and writing C/C++ code. While Java is much more convenient (due to portability reasons) for a Linux user like myself, I believe every programmer should know and understand how the underlying native system works. I've used C/C++ mostly for academic leaning exercises.

SQL

Essential in database work, I use SQL on a daily basis when doing web development. I'm as proficient writing queries as I am writing schemas, and have the most experience working with Postgres and MySQL.

HTML/SCSS

HTML and SCSS are my web development bread and butter. I commonly use another language to generate HTML strings, which I then style using SCSS. I have experience with both XHTML and HTML5.

Shell Scripts

A regular part of any Linux admin's daily routine, I've written shell scripts to do everything from routine server maintenance to conveniently wrapping more complex commands.

6502 Assembly

Just for fun. I enjoy learning about and writing 6502 assembly as a way to better understand how computers work under the hood. One day (when I have time) I want to write an NES game directly in 6502.
